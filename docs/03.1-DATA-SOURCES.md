### Setup-Time Derived Data (In-Memory)

These fields are built at `Game.setup()` from registry + config inputs.
They live in `G` and are immutable during gameplay. Moves never query
the registry — they use these pre-resolved fields.

| G field | Built from | WP | Purpose |
|---|---|---|---|
| `G.villainDeckCardTypes` | Registry card type classification | WP-014B | O(1) card type lookup during reveal |
| `G.cardStats` | Registry hero/villain stat values | WP-018 | Economy: attack, recruit, cost, fightCost |
| `G.heroAbilityHooks` | Registry hero card text parsing | WP-021 | Hero ability effect execution |
| `G.mastermind` | Registry mastermind + tactics data | WP-019 | Boss fight resolution |
| `G.matchConfiguration` | `MatchSetupConfig` (9 locked fields) | WP-005A | Match identity and composition |
| `G.hookRegistry` | Rule definitions (data-only) | WP-009A | Rule trigger/effect pipeline |
| `G.cardKeywords` | Registry villain/henchman ability text | WP-025 | Board keywords (Patrol/Ambush/Guard) per card |
| `G.schemeSetupInstructions` | Registry scheme data (MVP: `[]`) | WP-026 | Scheme board config; replay observability |

**Persistence note:** These fields are runtime-only. `G` is never
persisted to a database. Snapshots store zone counts only, never full
card arrays or derived fields. See ARCHITECTURE.md Section 3.

### PAR Artifacts

| Property | Value |
|---|---|
| **Source ID** | `par-artifacts` |
| **Local repo path** | `data/par/v{N}/scenarios/` (per-scenario JSON) |
| **Index** | `data/par/v{N}/index.json` (canonical oracle for PAR existence) |
| **Mutability** | Immutable — once published under `v{N}`, never modified |
| **Versioning** | Updates create new version directory (`v{N+1}`) |
| **Lifecycle** | Simulation (WP-049) -> write artifacts (WP-050) -> CI validation -> deploy |
| **Consumers** | Server only (competitive gate, leaderboard context) |
| **Engine usage** | None — engine never reads PAR artifacts |
| **Trust model** | Validated in CI only (`validateParStore`); server assumes pre-validated |
| **Integrity** | Hash-based; see `docs/12.1-PAR-ARTIFACT-INTEGRITY.md` |

### Match Setup Configuration

| Property | Value |
|---|---|
| **Source ID** | `match-setup-config` |
| **TypeScript contract** | `MatchSetupConfig` in `packages/game-engine/src/matchSetup.types.ts` |
| **Fields** | 9 locked fields (D-1244): `schemeId`, `mastermindId`, `villainGroupIds`, `henchmanGroupIds`, `heroDeckIds`, `bystandersCount`, `woundsCount`, `officersCount`, `sidekicksCount` |
| **Schema alignment** | JSON schema must match TypeScript contract exactly; `additionalProperties: false` enforced (D-1244 through D-1248) |
| **Mutability** | Immutable after match creation |
| **Lifecycle** | Client submits -> server validates -> passed to `Game.setup()` |
| **Trust model** | Validated by Zod at server boundary; engine may throw in `Game.setup()` on invalid config |
