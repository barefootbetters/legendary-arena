# WP-060 — Keyword & Rule Glossary Data Migration

**Status:** Ready for Implementation
**Primary Layer:** Content / Data (Registry)
**Schema Version:** 1
**Last Updated:** 2026-04-13
**Dependencies:** WP-003 (registry exists)

---

## Session Context

The old `modern-master-strike` project contains two authoritative glossary
files with human-readable definitions for all 102 game keywords and 18
game rules:

- `C:\Users\jjensen\bbcode\modern-master-strike\src\data\keywords-full.json`
- `C:\Users\jjensen\bbcode\modern-master-strike\src\data\rules-full.json`

These files were generated by `convert-keywords.mjs` from the `master-strike-data`
npm package. They contain structured definitions (paragraphs + bullet lists) and
flat summaries derived from the official Legendary Universal Rules v23 PDF.

Currently, the registry viewer hardcodes a subset of these definitions in
`apps/registry-viewer/src/composables/useRules.ts`. This packet migrates the
canonical data into `data/metadata/` so it is version-controlled, uploadable
to R2, and consumable by any future client or tool.

---

## Why This Packet Matters

1. **Registry viewer needs them** — keyword and rule tooltips are already
   displayed; the data should come from versioned JSON, not hardcoded strings
2. **Phase 5 will reference them** — WP-021 through WP-026 define keyword
   unions and hooks; having the glossary data in-repo prevents re-derivation
3. **Content authoring** — theme authors and future content tools need keyword
   definitions to validate and describe card abilities
4. **Single source of truth** — the data currently lives only in an external
   project with no git history

---

## Goal

Migrate `keywords-full.json` and `rules-full.json` into `data/metadata/`,
upload them to R2, and update the registry viewer to fetch them at runtime
instead of bundling hardcoded definitions.

### After completion:

- `data/metadata/keywords-full.json` exists (102 keywords with definitions)
- `data/metadata/rules-full.json` exists (18 rules with definitions)
- Both files are uploaded to R2 at `images.barefootbetters.com/metadata/`
- The registry viewer fetches glossary data from R2 at startup
- `useRules.ts` is simplified to consume the fetched data instead of
  hardcoding 200+ lines of definitions
- The `CLAUDE.md` for the registry viewer is updated

  truncate summaries, or flatten structure

**Session protocol:**

- Keep `parseAbilityText()` — unchanged
- Update `lookupKeyword()` and `lookupRule()` to query the fetched data

### E) Update `App.vue` to load glossary data on mount

Add `getKeywordGlossary()` and `getRuleGlossary()` calls in `onMounted`,
parallel with existing `getThemes()` call (non-blocking).
