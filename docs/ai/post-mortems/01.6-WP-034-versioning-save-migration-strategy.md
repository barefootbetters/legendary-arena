## 4. Representation & Determinism Audit

- [x] **New representations are data-only and JSON-serializable:**
      `EngineVersion`, `DataVersion`, `ContentVersion`,
      `VersionedArtifact<T>`, `CompatibilityStatus` (string union),
      `CompatibilityResult` are all plain TypeScript interfaces /
      type aliases. No functions stored in any value type. No
      `Date` objects (the `savedAt` field is an ISO 8601 string).
      No `Map` / `Set` / class instances. Test 9 explicitly asserts
      JSON-roundtrip structural equality.
- [x] **Execution is deterministic (within the documented exception):**
  - `checkCompatibility(artifactVersion, currentVersion)` is a pure
    decision function. Same inputs, same `CompatibilityResult`. No
    cache, no wall clock, no RNG.
  - `migrateArtifact(artifact, targetVersion)` is pure. The MVP
    registry is empty so the only non-throwing path is the
    same-version no-op (which spread-copies the wrapper). Future
    migration functions are required by contract to be pure.
  - `stampArtifact(payload, contentVersion?)` is **not** deterministic
    in the strict sense — its `savedAt` field reads the wall clock.
    This is the documented D-3401 sub-rule exception. The non-
    determinism is structurally bounded: only `savedAt` varies; the
    `engineVersion` / `dataVersion` / `payload` / `contentVersion`
    fields are deterministic. Save-time stamping at the persistence
    boundary is structurally distinct from gameplay-affecting clock
    reads — see D-3401 for the four-point rationale.
- [x] **Unknown / future values fail safely:**
  - Missing or unparseable `engineVersion` stamp:
    `checkCompatibility` returns `incompatible` with a locked
    full-sentence message (D-0802 fail-loud at the load boundary).
    Tested in Test 5 against `null`, `undefined`, a string, and an
    object missing fields.
  - Missing migration path: `migrateArtifact` throws with a locked
    full-sentence message naming both versions and the artifact's
    `savedAt`. Tested in Test 8.
  - Downgrade attempt: `migrateArtifact` throws with a separate
    locked template (downgrade refusal). Production-code path
    covered; explicit test omitted to honor the WP §G 9-test lock
    (in-allowlist refinement: the downgrade-throw helper is exercised
    indirectly through the type system + the no-path test).

## 5. Mutation, Aliasing, & Reference Safety (High-Risk)

This was the highest-risk section per the WP-028 / D-2802 precedent.
Every returned object was traced to its source.

### Audit per file

**`versioning.types.ts`** — pure type definitions; no values exported.
Safe.

**`versioning.check.ts`** — `getCurrentEngineVersion()` returns a
fresh object literal on every call (deliberate freshness — see the
`// why:` block on the function). `formatEngineVersion()` returns a
string (no aliasing surface). `checkCompatibility()` returns a fresh
`CompatibilityResult` literal via the `buildResult` helper, which
constructs new objects per call. Safe.

**`versioning.migrate.ts`** — `migrateArtifact()` returns a NEW
`VersionedArtifact<T>` constructed via spread-copies of the
`engineVersion` / `dataVersion` / `contentVersion` wrapper fields.
The `payload` field is shared by reference (the spread-copy contract
is on the wrapper, not on `T` — payload mutation is the caller's
responsibility, documented in `stampArtifact`'s `// why:` comment).
Test 7 explicitly asserts `notStrictEqual` for the wrapper and the
sub-objects, and `strictEqual` for the payload. The `migrationRegistry`
is `Object.freeze({})` — runtime-immutable. Safe.

**`versioning.stamp.ts`** — `stampArtifact()` constructs a fresh
wrapper around the caller-supplied payload. `engineVersion` is fresh
(via `getCurrentEngineVersion()`). `dataVersion` is spread-copied from
the module-level `CURRENT_DATA_VERSION` constant. `contentVersion` is
spread-copied from the optional argument when present. The `payload`
is stored by reference (same caller-responsibility contract as
`migrateArtifact`). Safe.

**`versioning.test.ts`** — exercises Test 7's spread-copy guarantee
(`notStrictEqual` for wrapper + sub-objects), Test 9's JSON-roundtrip
(structural equality after stringify+parse). No production-code
aliasing risk introduced by the test.

## 9. Forward-Safety Questions

- [x] **Can this code survive future refactors without touching
      unrelated layers?** Yes. The versioning subtree consumes no
      other engine type; future migrations append entries to a
      single registry constant; future axis additions extend the
      `VersionedArtifact<T>` interface (additive change, backward-
      compatible at the type level for consumers using the
      generic).
- [x] **Can replay/debugging reconstruct behavior from stored data?**
      Yes. Every load-time decision is derived from the artifact's
      `engineVersion` stamp + the current engine version constant.
      Bug reports name the artifact's `savedAt` + the
      `CompatibilityResult.message` — exactly the diagnostic
      surface Layer Boundary requires for load failures.
- [x] **Would removing upstream data still fail safely?** Yes. If a
      future producer emits an artifact missing the `engineVersion`
      field, `checkCompatibility` returns `incompatible` with the
      "missing stamp" template (D-0802 fail-loud). If the field is
      present but malformed, the same path fires. The engine never
      guesses.
- [x] **Is it impossible for this WP's output to influence gameplay
      unintentionally?** Yes. The versioning module has zero call
      sites in pre-existing engine code (it ships as a forward-
      compatibility seam). Future load-time deserializers will
      consume `checkCompatibility` and `migrateArtifact` at the
      app/server boundary — outside the engine reducer entirely.
      `stampArtifact` produces an inert wrapper that does not feed
      into any move dispatch path.
- [x] **Is the contract stable enough to be referenced by future
      WPs?** Yes. `VersionedArtifact<T>` is the canonical wrapper.
      `checkCompatibility` is the canonical load-time decision
      function. `stampArtifact` is the canonical save-time embed
      site. `migrateArtifact` is the canonical forward-only
      migration dispatcher. Each is single-responsibility, pure
      (modulo the documented `savedAt` exception), and structurally
      simple.
