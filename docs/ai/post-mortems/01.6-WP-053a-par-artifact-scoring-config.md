## Tradeoffs Observed

### `ParIndex` memory growth (Risk Awareness Note #2)

Inline `scoringConfig` materialization into `ParIndex.scenarios[key]` increases the per-scenario memory footprint by ~1 KB (a serialized `ScenarioScoringConfig`). For 100 scenarios this is ~100 KB of additional in-memory index size â€” negligible at MVP scale, but recorded here for future scale planning. If the index becomes operationally relevant at some future scale, the architecture supports a future cleanup WP that interns common config shapes (e.g., a config registry keyed by `scoringConfigVersion` with the index referencing by ID rather than inline). That cleanup is explicitly out of scope for WP-053a â€” D-5306b locks inline materialization for fs-free preservation.

### Artifact hash sensitivity (Risk Awareness Note #1)

Because `scoringConfig` is now embedded into both artifact shapes, even a key-ordering difference in the embedded JSON would alter `artifactHash`. The alphabetical-key rule (D-5306a) covers this in production output, and `canonicalJsonStringify` in `par.storage.ts` enforces sorted-key serialization for every write. Spot-checked during execution via the existing `writeSimulationParArtifact twice with identical input produces byte-identical file content` test, which now exercises the embedded-config path (after the fixture update). Determinism contract verified intact.

### Test fixture pre-existing under-spec discovered

The `createTestScoringConfig` factory in `par.storage.test.ts` previously had `bystanderReward: 50, villainEscaped: 300` â€” violating WP-048's `bystanderReward > villainEscaped` structural invariant. Pre-WP-053a this was harmless because the validator was never run against embedded artifact configs; WP-053a's new `validateParStore` extension exposed it. The factory weights were updated to `bystanderReward: 400` (still > villainEscaped: 300) as a mechanical fixture fix. No production code was affected; this was a test-only fixture under-spec that the new validator surface caught loudly at test time.

## Lessons Learned

1. **Sub-numbered EC casing convention should match precedent.** WP-007A/B established uppercase-letter-suffix convention. EC-053a (lowercase) created a hook-incompatibility surprise late in execution. Recommendation for future ECs: pick a single canonical casing (uppercase aligns with WP-007A/B precedent) and update the EC-TEMPLATE to lock it explicitly. The post-INFRA hook fix accepts both, so no immediate governance change is needed.

2. **Test fixtures are the most-affected surface in contract-extension packets.** Even though the `par.storage.test.ts` baseline used centralized factories, the +41-line fixture diff dwarfed the actual production-code changes. Calibration: future similar-scope contract-extension packets should pre-allocate ~30-50 lines of mechanical fixture work per affected test file.

3. **Layered defense is cheap and worthwhile even when redundant.** The gate's `assertEveryScenarioHasScoringConfig` guard is structurally unreachable through normal paths (the engine's shape validator catches first). It is preserved as defense-in-depth for future code paths that might construct a `ParIndex` programmatically. The cost is ~15 LOC; the benefit is one extra layer of "no partially-armed state" enforcement that cannot be circumvented by bypassing the loader.

4. **Browser-bundling friction is a real consideration for engine-package additions.** The new `scoringConfigLoader.ts` triggered Vite externalization warnings when the arena-client transitively imports the engine barrel. Resolved by switching the loader's `node:*` imports from named to namespace form (`import * as fsPromises from 'node:fs/promises'`); tree-shaking eliminates the unused module from the browser bundle. Recommendation: future engine-package additions that import `node:*` should default to namespace imports for consistency.

5. **The pre-commit review caught zero blocking issues.** The structural invariants were enforced by construction throughout execution; the review's role was to certify the contract surface, not to find bugs. Right-sized for a contract-extension packet of this scope.
