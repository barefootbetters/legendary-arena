## Section 9 — Lessons Learned

### 9.1 Authority-chain completeness is load-bearing

The v1 pre-flight caught three blockers that all traced to the same root cause: `INCIDENT_RESPONSE.md` and `ops.types.ts` were not named in the pre-Path-A WP-039's Context (Read First), so the authoring pass did not re-check the landed types before proposing parallel ones. The Path A rewrite added both documents to Context (Read First) as **AUTHORITATIVE** (caps-tagged), and the executing session re-read both before writing a single line of `LIVE_OPS_FRAMEWORK.md`. The discipline is: every WP that touches an existing authority surface must name that surface in Context (Read First) with an AUTHORITATIVE callout — missing this is a construction-time drift class that is expensive to unwind later.

### 9.2 Cross-link pointer is sufficient; restatement is drift

The temptation inside a WP-039-class document is to restate the severity table or the counter-field list "for reader convenience." The v1 pre-flight caught this as drift risk; the Path A rewrite replaced restatement with cross-link pointers plus one-line summaries. Verification Step 9 (grep for "Immediate rollback") returned zero — the severity table was not restated. Verification Step 5 (grep for parallel type names) returned zero after one false positive was resolved pre-Commit-A (see §10 below). The discipline is: for any surface that exists, write the pointer, not the restatement.

### 9.3 Grep guards catch what prose-review misses

The six verification-step greps in the session prompt (Steps 5, 9, 10, 11) caught one false-positive during authoring that prose-review missed: the sentence "Any future proposal to typify them (e.g., introduce a `MetricCategory` union)" contained the forbidden token `MetricCategory` inside meta-prose stating that no such union exists. The grep does not distinguish meta-prose from advocacy. The pre-Commit-A fix paraphrased the sentence to "Any future proposal to typify them into a code-level union is a separate pre-flight and requires its own DECISIONS.md entry" — same meaning, zero grep match. The discipline is: when a verification-step grep fires, paraphrase rather than excuse.
