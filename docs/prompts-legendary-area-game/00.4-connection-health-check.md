# Prompt 00.4 — Connection & Environment Health Check

> **FULL CONTENTS MODE** — Output: (1) files changed, (2) full contents of every
> changed file, (3) commands to run with expected output, (4) acceptance checklist.
> Never output diffs. Read-only diagnostics only — do not modify any external service.
> ESM only. Node v22+. Never print secrets.

## Purpose

Pre-flight diagnostic. Run on a fresh machine, a new developer setup, or before
any deployment troubleshooting. Verifies every external service, CLI tool, and
environment variable the project depends on is reachable and correctly configured —
before any code tries to use them.

Run this BEFORE Prompt 00.5 (R2 validation) and before any other prompt.

## Assumes

- Developer is on a Windows machine running PowerShell 7+
- Node.js v22+ is installed
- pnpm is installed globally
- The monorepo exists at `C:\pcloud\BB\DEV\legendary-arena`
- A `.env` file exists at the monorepo root (copied from `.env.example`)
- rclone is installed and configured with an R2 remote named `r2`
- GitHub repo: `https://github.com/barefootbetters/legendary-arena`

## Role

You are a senior DevOps engineer writing a developer environment health check
for a Node.js monorepo on Windows. The script must be honest — it reports
exactly what works and what doesn't, with clear remediation steps for every
failure mode. It makes no assumptions about what is already configured.

## Deliverables

### 1. Health check script (`scripts/check-connections.mjs`)

A standalone Node.js ESM script that:

1. Loads `.env` using Node.js v22's built-in `--env-file` flag (invoked via
   `package.json` script — no `dotenv` package import needed in this file)
2. Runs independent checks concurrently; runs env-dependent checks (DB, HTTP)
   after the env var check passes
3. Never throws unhandled exceptions — catches every failure and reports it
4. Prints a live status line as each check completes (not batched at end)
5. Prints a final summary report
6. Exit code 0 if all checks pass; exit code 1 if any check fails

#### Note on dotenv-cli vs Node --env-file

These are two distinct tools that serve different purposes:

- **`node --env-file=.env`** — Node v22 built-in flag. Used in `package.json`
  scripts to inject `.env` variables when running scripts directly. No package
  install required.
- **`dotenv-cli`** (`dotenv` binary) — A global CLI tool. Used in scripts where
  the `node --env-file` flag is unavailable (e.g., `pnpm exec` chains, shell
  scripts, PowerShell). Install with: `npm install -g dotenv-cli`

Both are checked. The health check itself uses `--env-file`. Other project
scripts (migrate, seed, validate) may use `dotenv-cli` as fallback.

#### Check implementations

**Node.js version check**
- Read `process.version`
- Fail if major version < 22
- Remediation: `Install Node.js v22+ from https://nodejs.org`

**pnpm version check**
- Run `pnpm --version` via `child_process.execSync`
- Fail if not found or version < 8
- Remediation: `npm install -g pnpm`

**dotenv CLI check**
- Run `dotenv --version` via `child_process.execSync`
- Fail if not found (dotenv-cli is required for scripts that cannot use `--env-file`)
- Fail if version < 16.x
- Also verify `.env` is syntactically parseable: run `dotenv -e .env -- node -e ""`
  and check for non-zero exit code (catches BOM encoding, unquoted special chars,
  missing `=` signs)
- Remediation: `npm install -g dotenv-cli`

**boardgame.io npm package check**
- Read `node_modules/boardgame.io/package.json` using `fs.readFileSync`
  (do not shell out — faster and more reliable)
- Fail if file does not exist: `pnpm install` not yet run
- Verify version is `0.50.x` — fail if major.minor is not `0.50`
  (boardgame.io has breaking API changes between minor versions)
- Also verify server entrypoint exists:
  `node_modules/boardgame.io/dist/cjs/server.js`
- Remediation: `pnpm install` or `pnpm add boardgame.io@0.50`

**`.env` file check**
- Verify `.env` exists at the project root
- Verify it is NOT byte-for-byte identical to `.env.example`
- Parse the file and flag values matching placeholder patterns:
  `your-*`, `change-me`, `REPLACE_*`, `<...>`, or empty string
- Fail if `.env` is missing entirely
- Warn (not fail) if placeholder values found

**Environment variable completeness check**
- Check each var in the REQUIRED_VARS map is present in `process.env` and non-empty
- Group output by service
- Fail if any required variable is missing

**PostgreSQL connection check**
- Connect using `DATABASE_URL` via `pg` pool
- Run `SELECT current_database(), version()`
- Verify database name matches `process.env.EXPECTED_DB_NAME`
- Disconnect cleanly after check
- Fail on connection error — include pg error code and remediation hint
- Timeout: 5 seconds

**boardgame.io server health check**
- GET `${GAME_SERVER_URL}/health`
- Expect HTTP 200 and JSON body `{ status: 'ok' }`
- Fail if unreachable, wrong status, or response > 5 seconds
- Remediation: `Is the Render service running? Check https://dashboard.render.com`

**Cloudflare R2 public URL check**
- GET `${R2_PUBLIC_URL}/registry-config.json`
- Expect HTTP 200 and `content-type: application/json`
- Do not parse body — verify reachability and content type only
- Timeout: 5 seconds

**Cloudflare Pages check**
- GET `${CF_PAGES_URL}` (value from env)
- Expect HTTP 200
- Timeout: 5 seconds
- Remediation: `Check Cloudflare Pages dashboard for deployment status`

**GitHub reachability check**
- GET `https://api.github.com/repos/barefootbetters/legendary-arena`
  (unauthenticated — public repo)
- Expect HTTP 200 and `{ name: 'legendary-arena' }`
- Run `git remote get-url origin` via `child_process.execSync`
- Verify local remote URL matches the expected GitHub repo URL
- Fail if remote is misconfigured or GitHub is unreachable

**rclone check**
- First verify rclone config file exists at Windows path:
  `%APPDATA%\rclone\rclone.conf` (expand via `process.env.APPDATA`)
  Warn if missing before attempting CLI commands
- Run `rclone version` via `child_process.execSync` — verify binary on PATH
- Run `rclone lsd r2:` to list the bucket root
- Verify output contains `metadata` folder and at least one set abbreviation folder
- Fail if rclone not found, not configured, or cannot list bucket
- Remediation: `rclone config — see docs/rclone-setup.md`

### 5. PowerShell quick-check (`scripts/Check-Env.ps1`)

A self-contained PowerShell 7+ script. Requires no Node.js and no `.env`.
Run this first on any new machine before anything else.

**Section A — PATH tool verification**

Use `Get-Command` (not `where.exe`) for each tool. `Get-Command` is the
PowerShell-native approach: searches PATH correctly on Windows, handles both
`.exe` and `.cmd` extensions, returns a structured object with resolved path.

Check these tools are on PATH:

| Tool command | Min version | Remediation if missing |
|---|---|---|
| `node` | v22 | https://nodejs.org |
| `npm` | any | bundled with Node.js |
| `pnpm` | v8 | `npm install -g pnpm` |
| `dotenv` | v16 | `npm install -g dotenv-cli` |
| `git` | any | https://git-scm.com |
| `rclone` | any | https://rclone.org/downloads |

For each tool:
1. `Get-Command <tool> -ErrorAction SilentlyContinue`
2. If `$null`: print `✗ <tool> : NOT FOUND on PATH` + remediation
3. If found: run `& <tool> --version` (or `node --version`, `git --version`)
   to get version string, print `✓ <tool> : vX.Y.Z  (<resolved path>)`
4. Parse version string and warn if below minimum

Print the full PATH at the top of Section A — critical for debugging
"not found" failures caused by stale or shadowed install locations:

```powershell
Write-Host "`nPATH entries searched:" -ForegroundColor DarkGray
$env:PATH -split ';' | Where-Object { $_ } | ForEach-Object {
    Write-Host "  $_" -ForegroundColor DarkGray
}
```

**Section B — `.env` file check**

- `Test-Path .env` — warn (not fail) if missing; may not exist on fresh setup
- If found: load with `Get-Content .env`, check each `KEY=VALUE` line
  - Skip blank lines and `#` comments
  - Print each key name and SET/EMPTY/PLACEHOLDER status (mask actual values)
  - Flag values matching: `your-*`, `REPLACE_*`, `change-me`, `<...>`, or empty

**Section C — rclone config check**

- Check for rclone config at Windows path: `$env:APPDATA\rclone\rclone.conf`
- Warn clearly if missing (rclone may be installed but not yet configured)
- If found: read with `Get-Content`, verify an `[r2]` remote section exists

**Section D — npm package check (boardgame.io)**

- `Test-Path node_modules\boardgame.io\package.json`
- If found: `Get-Content | ConvertFrom-Json` → print version
- If not found: print remediation `pnpm install`

## Operational Notes (answer directly)

1. **Local vs Render env vars**: `DATABASE_URL` is auto-wired by Render but
   must be set manually in `.env` for local dev. What is the safest way to
   get a local connection string without pointing developer machines at the
   production Render database?

2. **`PORT` in Render dashboard**: Render injects `PORT` automatically. What
   breaks if a developer accidentally adds `PORT=8000` to the Render dashboard
   env vars manually?

3. **rclone config on Windows**: The rclone config file lives at
   `%APPDATA%\rclone\rclone.conf` on Windows (not `~/.config/rclone`).
   Show the PowerShell expression to resolve this path reliably across
   different Windows user account names.

4. **CI environment**: This script must NOT run in GitHub Actions — no rclone,
   no local `.env`, no Render server to ping. Show the exact `if:` condition
   to exclude this script from the CI workflow defined in Prompt 12.

## Hard Constraints

- ESM only — no CommonJS `require()`
- Node.js built-in `fetch` for HTTP checks — no axios, no node-fetch
- `pg` for database check only — no other database packages
- `child_process.execSync` for CLI tool checks — all wrapped in try/catch
- Read-only — no modification of any files, services, or configuration
- All network checks must enforce a 5-second timeout
- Script must not crash if `.env` is missing — handle gracefully and report
- Secrets must never be printed — mask all values in output
- rclone config path must use `process.env.APPDATA` — never hardcode a username
